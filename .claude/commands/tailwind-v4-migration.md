  "content": "# Tailwind CSS v4 Migration & Troubleshooting\n\nHelp with Tailwind CSS v4 specific issues and migration from v3.\n\n## Common v4 Issues and Solutions\n\n### Issue 1: Unknown Utility Class Errors\n\n**Symptom:**\n```\nCannot apply unknown utility class `bg-background`\nCannot apply unknown utility class `text-foreground`\n```\n\n**Root Cause:** \nIn Tailwind v4, custom utilities defined via @theme directive or CSS variables don't automatically generate utility classes.\n\n**Solution:**\n1. Explicitly define in tailwind.config.ts:\n```typescript\nexport default {\n  theme: {\n    extend: {\n      colors: {\n        background: 'var(--background)',\n        foreground: 'var(--foreground)',\n        primary: {\n          DEFAULT: 'var(--primary)',\n          foreground: 'var(--primary-foreground)',\n        }\n      }\n    }\n  }\n}\n```\n\n2. Define CSS variables:\n```css\n:root {\n  --background: white;\n  --foreground: #0f172a;\n  --primary: oklch(0.61 0.25 264);\n  --primary-foreground: white;\n}\n```\n\n### Issue 2: @theme Directive Misunderstanding\n\n**v3 Approach:**\n```css\n/* This would work in some setups */\n@layer base {\n  :root {\n    --background: theme('colors.white');\n  }\n}\n```\n\n**v4 Approach:**\n```css\n/* @theme doesn't create utilities */\n@theme {\n  --color-background: white;\n}\n\n/* Still need config for utilities */\n```\n\n### Issue 3: CSS Processing Changes\n\n**Check for:**\n- Vite with @tailwindcss/vite plugin (v4 recommended)\n- PostCSS config (may not be needed with Vite plugin)\n- Build tool compatibility\n\n### Migration Checklist\n\n#### 1. Update Dependencies\n```json\n{\n  \"devDependencies\": {\n    \"tailwindcss\": \"^4.0.0\",\n    \"@tailwindcss/vite\": \"^4.0.0\"\n  }\n}\n```\n\n#### 2. Update Vite Config (if using Vite)\n```typescript\nimport tailwindcss from '@tailwindcss/vite'\n\nexport default {\n  plugins: [tailwindcss()]\n}\n```\n\n#### 3. Review Custom Utilities\n- Find all custom utility usage\n- Add to theme.extend\n- Define CSS variables\n\n#### 4. Update CSS File\n```css\n/* v4 approach */\n@import \"tailwindcss\";\n\n/* Custom CSS variables */\n:root {\n  /* Define all custom properties */\n}\n```\n\n### Quick Fixes for Common Patterns\n\n#### Pattern: Semantic Colors\n**Before:** Using undefined utilities\n```jsx\n<div className=\"bg-background text-foreground\">\n```\n\n**After Option 1:** Use CSS variables directly\n```jsx\n<div style={{ \n  backgroundColor: 'var(--background)',\n  color: 'var(--foreground)'\n}}>\n```\n\n**After Option 2:** Configure properly\n```typescript\n// tailwind.config.ts\ncolors: {\n  background: 'var(--background)',\n  foreground: 'var(--foreground)'\n}\n```\n\n#### Pattern: Dark Mode\n**Before:** Might not work\n```css\n@media (prefers-color-scheme: dark) {\n  :root {\n    --background: black;\n  }\n}\n```\n\n**After:** Proper dark mode\n```css\n:root {\n  --background: white;\n}\n\n.dark {\n  --background: #0f172a;\n}\n```\n\n### Validation Steps\n\n1. **Check all className usage** for undefined utilities\n2. **Verify tailwind.config.ts** has all custom colors\n3. **Ensure CSS variables** are defined\n4. **Test build process** works without errors\n5. **Verify dark mode** if applicable\n\n### Tools & Commands\n\n```bash\n# Find undefined utilities\ngrep -r \"className=\" src/ | grep -E \"(bg|text|border)-(background|foreground|primary)\"\n\n# Check Tailwind version\nnpm list tailwindcss\n\n# Validate config\nnpx tailwindcss init --help\n```\n\n## When to Use This Guide\n\n- Seeing \"Cannot apply unknown utility class\" errors\n- Migrating from v3 to v4\n- Custom theme not working as expected\n- CSS variables not generating utilities\n- Build errors after Tailwind upgrade"
