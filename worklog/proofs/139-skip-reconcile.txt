## Fix-Summary

ClubProvider.tsx — conditional reconcile:
- Before: `if (inflightRef.current.size === 0) { reconcile }`
- After:  `if (currently && inflightRef.current.size === 0) { reconcile }`
  (`currently=true` = unfollow path; `currently=false` = follow path → skip)

Rationale:
- Follow: Optimistic state is complete (we know clubData). Server-reconcile adds no
  information, only risks pgBouncer read-after-write transient.
- Unfollow: Service-layer transfers primary-flag to unpredictable next-club. Reconcile
  is the only way to learn the new primary ordering.

Plus beforeEach fix: mockReset() for leaky mockResolvedValueOnce queues between tests.
