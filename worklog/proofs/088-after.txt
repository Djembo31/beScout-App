## Sentry-Flow in Production

1. User triggert AuthProvider.loadProfile fallback
2. Eine der 3 Queries (getProfile / getPlatformAdminRole / getClubAdminFor) timeouts oder wirft
3. allSettled resolved mit `{status: 'rejected', reason: Error}` für diesen Slot
4. `logSilentRejects` iteriert results, findet rejected slot
5. Im Prod: `Sentry.captureException(err, { tags: {silentReject: 'true', label: 'AuthProvider.loadProfile'}, extra: {index, totalResults} })`
6. Sentry Dashboard zeigt Event mit Tag `silentReject:true` + `label:AuthProvider.loadProfile` — filterbar, gruppierbar per Stacktrace
