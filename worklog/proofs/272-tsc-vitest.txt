4) Root-Cause-Analyse

4 unabhängige Pfade die Duplicate-State erzeugen können:

PATH 1: lineupStore.selectPlayer
  Pre-Slice-272:
    selectedPlayers: [
      ...state.selectedPlayers.filter((p) => p.slot !== slot),  // ← nur Slot
      { playerId, position, slot, isLocked: false },
    ]
  Bug: Player X auf Slot 0, dann selectPlayer(X, 1) → X auf BEIDEN Slots

PATH 2: lineupStore.setBenchSlot
  Pre-Slice-272: dedupted INNERHALB Bench, nicht vs. Starter
  Bug: Player X auf Starter-Slot 0, dann setBenchSlot('bench_o1', X)
       → X erscheint im Starter UND auf Bench-O1

PATH 3: useLineupBuilder.getAvailablePlayersForPosition
  Pre-Slice-272:
    const usedIds = new Set(selectedPlayers.map((p) => p.playerId));
    // ← nur Starter, nicht Bench
  Bug: Bench-Spieler erscheint im Starter-Picker → wählbar → Duplicate

PATH 4: LineupPanel:854-865 Quick-Add (Holdings-Strip-Click)
  Pre-Slice-272:
    const freeSlot = formationSlots.find(slot =>
      slot.pos === player.pos &&
      !selectedPlayers.some(sp => sp.position === slot.pos && sp.slot === slot.slot),
    );
    if (freeSlot) onSelectPlayer(player.id, ...);
  Bug: Klick auf Player im Holdings-Strip findet nächsten freien Slot UND fügt Player
       hinzu, auch wenn Player schon im Lineup ist

5) Defense-in-Depth Fix
═══════════════════════════════════════════════════════════

Store-Layer (selectPlayer):
  selectedPlayers: [
    ...state.selectedPlayers.filter((p) => p.slot !== slot && p.playerId !== playerId),
    { playerId, position, slot, isLocked: false },
  ],
  + benchClear: entfernt Player aus allen Bench-Slots beim Starter-Promote

Store-Layer (setBenchSlot):
  + nextStarter: entfernt Player aus Starter-Slots beim Bench-Set

Hook-Layer (getAvailablePlayersForPosition):
  if (benchGk) usedIds.add(benchGk);
  if (benchO1) usedIds.add(benchO1);
  if (benchO2) usedIds.add(benchO2);
  if (benchO3) usedIds.add(benchO3);

UI-Layer (LineupPanel Quick-Add):
  if (selectedPlayers.some(sp => sp.playerId === player.id)) return;
