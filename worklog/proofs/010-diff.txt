DRIFT CLASS THIS GUARD CATCHES
=============================================================================
Developer adds `throw new Error('someNewKey')` in a service, forgets to add
`someNewKey` to KNOWN_KEYS (errorMessages.ts) or ERROR_MAP. Any consumer
using `setError(mapErrorToKey(normalizeError(err)))` would silently fall
through to 'generic' → user sees `errors.generic` instead of the intended
translated message.

CI test now fails on that drift with a pointer to file:line + the key.
Resolver path:
  (a) Add key to KNOWN_KEYS + translations in messages/de.json + messages/tr.json
      (CEO-Scope for new user-facing text), or
  (b) Add WHITELIST entry with a citation to the explicit consumer-side
      handler that resolves the key without going through mapErrorToKey
      (e.g. fantasy-namespace keys handled in useEventActions.ts).
