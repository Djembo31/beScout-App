PHASE 2 LIVE-LEHRE — Catch-22 entdeckt
==================================================

Direkt nach enforce_admins=true PUT:
$ git push origin main
remote: - 4 of 4 required status checks are expected.
 ! [remote rejected]   main -> main (protected branch hook declined)

ROOT-CAUSE: enforce_admins=true UND strict=true UND required_status_checks
sind für PR-Merge-Workflow designed, NICHT für Solo-Dev direct-push:

  - GitHub fordert dass NEUER tip-commit alle 4 grünen Status-Checks hat
  - CI startet erst NACH Push (post-push-Event)
  - Catch-22: Push blockiert bis CI grün, CI grün braucht Push.

BeScout fährt aber direct-push (242 Slice-Historie). enforce_admins=true
würde Workflow auf PR-für-alles zwingen — 10× Friktion pro Slice, Solo-Dev-
Anti-Pattern.

==================================================
ANIL-DECISION: Option C
==================================================

enforce_admins=false zurück (Status-Quo-pre-Slice-244-Phase-2 für
direct-push) ABER:

  → Slice 248 NEU — Pre-Push-Hook der lokal alle 4 Status-Checks
    simuliert (audit:type-truth + audit:stale + audit:wiring +
    tsc + vitest + bundle-budget-check). Echte Sicherheit ohne
    Workflow-Bremsung.

Trade-Off: Pre-Push-Latenz ~60s (akzeptabel weil bewusst-vor-Push).
Architektonische Lösung statt Convention.
