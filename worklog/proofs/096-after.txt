## Sentry-Event-Flow (nach Deploy)

1. User signed in → `Sentry.setUser({id: 'uuid'})` + Breadcrumb
2. User triggert Error → Event gesendet mit user.id + letzte Breadcrumbs (inkl. signed_in)
3. Dev sieht Sentry-Event: welcher User (via UUID), in welcher Session-Phase (Breadcrumb-Trail)
4. GDPR-audit-safe: kein email/handle/name in Events
