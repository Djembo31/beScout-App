## CI-Flow
1. PR commit
2. GitHub Actions `lint` job: `npm ci` → `lint` → `type-check` → `audit:silent-fail:check`
3. HIGH-increase → exit 1 → PR red
4. Dev muss entweder (a) neue silent-fails fixen ODER (b) baseline explicit updaten mit Begruendung im PR

## Baseline-Update-Workflow
- Silent-Fail gefixt → `npm run audit:silent-fail` → counts ansehen
- Wenn gesunken: `.audit-baseline.json` updaten auf neue niedrigere counts → commit
- Verhindert "baseline stays same forever, fixes don't lower bar"
